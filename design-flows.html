2Partner dashboard & warehouse selection

Scopes every partner-side list and stat · two-step partner → warehouse picker

Flow diagram

flowchart TD A([Enter as Partner role]) --> B{Partner & warehouse selected?} B -- No --> C[Open PartnerWarehouseSelector sheet] B -- Yes --> D[PartnerDashboard] C --> C1[Step 1: List partners] C1 --> C2[Step 2: List warehouses for chosen partner] C2 --> C3{Warehouse has data?} C3 -- No --> C4[Empty state in warehouse list] C3 -- Yes --> C5[Confirm selection] C5 --> D D --> D1[Header: current partner · warehouse · selector button] D --> D2[Stat cards: pending / draft / registered / in-transit] D --> D3[Recent orders list] D --> D4[Quick action tiles] D4 --> E1[Create Order] D4 --> E2[View Orders] D4 --> E3[View Boxes] D4 --> E4[View Deliveries] D4 --> E5[View Returns] D4 --> E6[Reports] D1 -. tap selector .-> C

States & decisions

  • No selection — selector sheet auto-opens on first visit; dashboard stats hidden.
  • Selected — dashboard stats, recent orders and lists are filtered to the chosen partner + warehouse.
  • Re-select — tap selector → can go back from warehouse step to partner step without losing dashboard state.
  • Partner = Thrifted — "Create Order" opens the ThriftedOrderCreationScreen (manual/bulk). Other partners see "not available" placeholder.
  • Partner = Sellpy Operations — orders arrive via API; "Create Order" is hidden in favour of "View Orders".

flowchart TD E1[PostRegistrationDialog: 'Create delivery note'] --> DNC E2[Registered order · OrderShipmentDetailsScreen 'Create delivery note'] --> DNC DNC[DeliveryNoteCreationScreen] DNC --> INFO[Order info card: total · assigned · unassigned · boxes] DNC --> ADDBOX{Add box} ADDBOX -- Scan label --> SB[ActiveScanner · box mode] SB --> SB1{Unique label?} SB1 -- No --> SBERR[Toast: duplicate] SBERR --> ADDBOX SB1 -- Yes --> BOXADDED[Box card created · status=pending] ADDBOX -- Enter manually --> EM[Dialog · type label] EM --> SB1 BOXADDED --> ITEMS{Add items to box} ITEMS -- Scan item --> SI[ActiveScanner · item mode] SI --> SI1{Item in order & unassigned?} SI1 -- No --> SIERR[Toast: not in available items] SIERR --> ITEMS SI1 -- Yes --> ASSIGNED[Item moves into box] ITEMS -- Add manually --> MAN[Dialog · multi-select unassigned items] MAN --> ASSIGNED ASSIGNED --> UNALERT{Any unassigned left?} UNALERT -- Yes --> UNWARN[Red alert card · up to 5 shown] UNALERT -- No --> READY[Ready to register] UNWARN --> ITEMS READY --> REG{Register delivery} REG -- Empty box exists --> ERR1[Error: empty boxes] REG -- Unassigned items --> ERR2[Error: items unassigned] REG -- OK --> SUCCESS[DeliveryNote status=registered · Order status=in-transit · back to PartnerDashboard] ERR1 --> ITEMS ERR2 --> ITEMS

Validation rules

  • At least one box exists
  • No empty boxes
  • Every order item is assigned to exactly one box
  • No item can be in multiple boxes
  • All items must have retailerItemId (Sellpy: enforced; Thrifted: entered at creation)

Key UI elements

  • StockCheckScreen: tabs (Not scanned · Scanned) · sticky active scanner · counter chips
  • StockCheckReviewScreen: tabs (Not scanned · Not found · Scanned · All included) · per-item status selector
